مسیر یک درخواست وب

از تایپ آدرس تا نمایش صفحه — چه اتفاقی می‌افتد؟

مسیر درخواست

مرورگر آدرس را تجزیه می‌کند: پروتکل (http/https)، نام دامنه و مسیر — همان «کجا برو»ی درخواست تو.
→
مرورگر از سرور DNS می‌پرسد IP مربوط به نام دامنه چیست. پاسخ اغلب در کَش ذخیره می‌شود تا بار بعد سریع‌تر باشد.
→
با Three-Way Handshake (SYN → SYN-ACK → ACK) کانال قابل‌اعتماد TCP بین مرورگر و سرور ساخته می‌شود.
→
در HTTPS، گواهی سرور بررسی می‌شود و کلیدهای رمزنگاری مبادله می‌شود تا کسی میان‌راه نتواند محتوا را بخواند.
→
درخواست HTTP شامل متد (GET/POST)، هدرها (مثل User-Agent) و در صورت نیاز بدنه پیام است — همه روی همان اتصال امن ارسال می‌شود.
→
سرور کد وضعیت (مثل 200) و بدنه پاسخ را برمی‌گرداند. مرورگر HTML را parse می‌کند، JSON را پردازش می‌کند یا فایل را ذخیره و نمایش می‌دهد.

محتوا تغییر نکرده، از کَش بخوان

401

باید وارد شوی

403

صفحه پیدا نشد

این متد HTTP مجاز نیست

500
Internal Server Error

خطای داخلی سرور

502

سرور واسط پاسخ نگرفت

سرور موقتاً در دسترس نیست

504

وقتی کار نمی‌کند

DNS Error

دامنه پیدا نشد.

🔧 راهنمای دیباگ: آدرس URL را بررسی کن و مطمئن شو اینترنت وصل است.

Timeout

سرور در زمان مناسب پاسخ نداد.

🔧 راهنمای دیباگ: اتصال شبکه را چک کن یا چند لحظه بعد دوباره تلاش کن.

404 Not Found

فایل یا صفحه پیدا نشد.

🔧 راهنمای دیباگ: مسیر فایل (href، src یا URL) را بررسی کن و مطمئن شو فایل وجود دارد.
