# مسیر یک درخواست وب 🌐

یک صفحه وب آموزشی تک‌فایلی که مسیر کامل یک درخواست HTTP را از تایپ آدرس تا نمایش صفحه به‌صورت تصویری و تعاملی نشان می‌دهد.

---

## پیش‌نمایش

صفحه شامل سه بخش اصلی است:

- **مسیر درخواست** — دیاگرام گام‌به‌گام: `URL ← DNS ← TCP ← TLS ← HTTP ← Response`
- **کدهای وضعیت HTTP** — گرید رنگ‌بندی‌شده برای کدهای 2xx، 3xx، 4xx و 5xx
- **وقتی کار نمی‌کند** — راهنمای دیباگ برای خطاهای رایج

## مراحل مسیر درخواست

| مرحله | توضیح کوتاه |
|---|---|
| 🔗 URL | آدرسی که کاربر در مرورگر وارد می‌کند |
| 🌐 DNS | تبدیل نام دامنه به آدرس IP |
| 🤝 TCP | برقراری اتصال پایدار (Three-Way Handshake) |
| 🔒 TLS | رمزنگاری ارتباط برای HTTPS |
| 📤 HTTP | ارسال درخواست به سرور |
| 📥 Response | دریافت پاسخ از سرور |

روی هر مرحله در صفحه کلیک کنید تا توضیح کامل‌تر نمایش داده شود.

---

## کدهای وضعیت HTTP پوشش‌داده‌شده

- **2xx موفق:** 200، 201، 204
- **3xx ریدایرکت:** 301، 302، 304
- **4xx خطای کاربر:** 400، 401، 403، 404، 405، 429
- **5xx خطای سرور:** 500، 502، 503، 504

---

## خطاهای رایج و دیباگ

| خطا | علت | راه‌حل |
|---|---|---|
| DNS Error | دامنه پیدا نشد | URL را چک کن، اینترنت را بررسی کن |
| Timeout | سرور پاسخ نداد | شبکه را چک کن، بعداً دوباره تلاش کن |
| 404 Not Found | فایل یا صفحه وجود ندارد | مسیر `href`، `src` یا URL را بررسی کن |
